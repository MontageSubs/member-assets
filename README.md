# member-assets

欢迎来到蒙太奇字幕社区的成员身份中心！

member-assets 不仅仅是一个元数据数据库，它是我们每位成员在社区中的“数字名片”。在这里，我们关联昵称、GitHub 账号与头像，并记录每一位贡献者的足迹与成就，让社区的每一份付出都被看见。

> [!NOTE]
> 本指南基于 V1 API 版本，旨在帮助你快速构建并维护自己的社区档案。

## 仓库结构

为了保证系统的稳定运行，我们将“逻辑”与“数据”分开存放：

- **[main 分支](../../tree/main)**：这里的代码是整个系统的“发动机”，负责处理所有自动化维护逻辑（当前 API 版本为 V1）。
- **[v1 分支](../../tree/v1)**：这是我们的“档案库”，存放着所有成员的公开个人资料。你的昵称、ID、贡献记录和成就都记录在这里。
- **[Releases 页面](../../releases)**：这是头像的“分发站”，提供多种尺寸和版本的静态托管，确保你的头像在任何地方都能快速加载。

## 新成员入驻

首先欢迎你成为社区的一员！如果你还没有 GitHub 账号，可以先创建一个，也可以在社区群组中提问，大家会非常热情地引导你完成。

创建好账号后，请参考 [加入社区] 文档 （当前文档暂不可用，你可以联系 小p 协助你加入）申请加入。一旦通过，你就可以开始打造你的专属个人资料了。

### 创建你的个人资料

1. **触发创建**：前往 **[v1-create-profile.yml](../../actions/workflows/v1-create-profile.yml)** $\rightarrow$ 点击右侧 **Run workflow**。
2. **定义身份**：输入你的 **昵称** 以及 **GitHub 用户名**。
   
 > [!TIP]
 > 强烈建议关联 GitHub！这样当你为社区贡献代码或翻译时，机器人能精准地识别你的身份，并将贡献记录自动同步到你的个人资料中。

3. **查看名片**：等待 10-20 秒后，前往 **[v1 分支](../../tree/v1)**。在成员列表中找到你的文件夹，点击进入，你的第一份社区档案就诞生了！

## 个人资料管理

### 如何让你的个人资料更出彩？

你的个人资料页是你向社区其他成员展示自己的窗口。

1. 前往 **[v1 分支](../../tree/v1)**，找到你的文件夹并进入页面。
2. 点击右上角的 **笔形 🖊️ 编辑按钮** 修改 README。
3. **编辑指南**：
   - **基础信息**：请在 `profile:input` 标记的区块内填写。
   - **个性化展示**：你可以自由使用 Markdown 编写你的荣誉、成就或个人寄语。如果编写过程中遇到问题，欢迎在社区群组中求助，成员们会协助你完成。

> [!WARNING]
> **请保护好 HTML 代码**：请勿修改文档中的 HTML 标签或复杂代码块。这些是机器人渲染页面的“骨架”，一旦损坏，你的名片可能会无法正常显示。

**维护小贴士：**
- **自动同步**：头像、简介和社区贡献由机器人默默为你维护。
- **手动记录**：目前的“成就”部分需要你手动填写，记得及时更新你的里程碑！
- **保存生效**：点击 **Commit changes...** 保存，十几秒后，机器人会将你的修改同步为机器可读格式。

### 关于自定义头像

虽然机器人会自动同步你的 GitHub 头像，但你也可以上传一个更符合社区氛围的自定义头像：

1. 在你的个人资料页面，点击头像下方的 **[添加自定义头像]** 按钮。
2. 上传图片（支持 png, jpg, gif, webp）。
   - **建议**：文件名设为 `avatar`，尺寸 500px 以上且为方形（非方形图片将被自动居中裁切）。

**更新头像**：若需更换，请前往 `https://github.com/MontageSubs/member-assets/tree/v1/members/<你的社区用户ID>/avatar` 文件夹，替换旧文件即可。

## 常见问题

### 我的 GitHub 头像更新了，为什么这里还没变？

为了节省资源，自动同步每周运行一次。如果你急于展示新头像，可以手动触发：
- 前往 **[v1-sync-avatars.yml](../../actions/workflows/v1-sync-avatars.yml)** $\rightarrow$ 点击 **Run workflow**。
- 输入你的 **社区成员 ID** 即可进行针对性更新。

### 什么是社区成员 ID？

社区成员 ID 是一串随机字符，它是你在本数据库中的“唯一身份凭证”。
- **不可更改**：无论你以后如何更改昵称或 GitHub 账号，这个 ID 始终不变。
- **核心作用**：它像一把钥匙，确保你在参与不同项目时的所有贡献记录都能准确地关联到同一个人身上。

### 我可以删除我的信息吗？

可以。我们尊重每位成员的隐私选择。如果你决定离开社区，可以通过以下方式注销：
1. 前往 **[v1-delete-profile.yml](../../actions/workflows/v1-delete-profile.yml)** $\rightarrow$ 点击 **Run workflow**。
2. **输入两次你的社区 ID** 以确认删除。

> [!CAUTION]
> **注销前请留意**：
> - 删除后，你的贡献页面将失效，他人访问时会看到失效的链接或头像。
> - 已发布在作品中的昵称无法撤回。
> - **温馨建议**：如果你只是想暂时低调，我们更建议你通过更改昵称、取消关联 GitHub 或更换头像来保护隐私。
> - **底线**：Action 权限虽然开放，但请尊重他人，禁止尝试删除其他成员的信息。
